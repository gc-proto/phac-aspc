Bar graph

A bar graph is a visual representation of data that uses bars to show the relative values of different data points.

Bar graphs can be oriented either horizontally or vertically.

On this page

  • When to use
  • What to avoid
  • Design and code
  • Content and design guidelines
  • Accessibility guidelines

When to use

  • To show the relative size or value of different data points
  • To compare the values of different data points
  • To show the distribution of data over a range of values

Examples

  • A bar graph comparing the vaccination rates of different age groups in a population
  • A bar graph showing the number of people who have been screened for a specific health condition, broken down by sex

When to use vertical bar graphs

  • You have a large number of data points or categories
  • You want to emphasize the y-axis values
  • You want to show the data in a compact, space-efficient way

When to use horizontal bar graphs

  • You have long category labels
  • You want to emphasize the x-axis values
  • You want to show the data in a way that is easy to read and comparable

When to use stacked bar graphs

  • The sum of the values of the different categories is important
  • You want to show both the total value and the distribution of the categories that make up that value

When to use grouped bar graphs

  • To show the relative size of different categories on the same X axis
  • When the sum of the values is not the focus

What to avoid

  • Don't use bar graphs to show data that does not have a clear numeric value
  • Don’t use bar graphs to show data with different units or different scales

function drawGraph(data) {

// List of subgroups = header of the csv files = soil condition here
var subgroups = data.columns.slice(1)

// List of groups = species here = value of the first column called group -> I show them on the X axis
var groups = d3.map(data, function(d){return(d.group)})

console.log(d3.map(data, function(d){return(d.group)}))
// Add X axis
var x = d3.scaleBand()
    .domain(groups)
    .range([0, width])
    .padding([0.2])
svg.append("g")
  .attr("transform", "translate(0," + height + ")")
  .call(d3.axisBottom(x).tickSize(0));

// Add Y axis
var y = d3.scaleLinear()
  .domain([0, 40])
  .range([ height, 0 ]);
svg.append("g")
  .call(d3.axisLeft(y));

// Another scale for subgroup position?
var xSubgroup = d3.scaleBand()
  .domain(subgroups)
  .range([0, x.bandwidth()])
  .padding([0.05])

// color palette = one color per subgroup
var color = d3.scaleOrdinal()
  .domain(subgroups)
  .range(['#e41a1c','#377eb8','#4daf4a'])

// Show the bars
svg.append("g")
  .selectAll("g")
  // Enter in data = loop group per group
  .data(data)
  .enter()
  .append("g")
    .attr("transform", function(d) { return "translate(" + x(d.group) + ",0)"; })
  .selectAll("rect")
  .data(function(d) { return subgroups.map(function(key) { return {key: key, value: d[key]}; }); })
  .enter().append("rect")
    .attr("x", function(d) { return xSubgroup(d.key); })
    .attr("y", function(d) { return y(d.value); })
    .attr("width", xSubgroup.bandwidth())
    .attr("height", function(d) { return height - y(d.value); })
    .attr("fill", function(d) { return color(d.key); });

}

Content and design guidelines

  • Always start the Y axis at 0
  • Consider ordering the bars by value
  • Use colour wisely - in general, use the same colour for all bars, and use colour to highlight a specific bar
  • Consider having the value directly within the bars
  • Design bars so that they are wider than the space between them
  • Follow the general design guidelines

Accessibility guidelines

  • Provide a data table with the graph
  • If the bar graph tells a clear story, include that story in the alt text - for example, if the graph shows that category has a much higher value than the others, spell that out in the alt text
  • Follow the general accessibility guidelines
