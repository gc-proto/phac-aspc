Bar graph

A bar chart is a visual representation of data that uses bars to show the relative values of different data points.

Bar charts can be oriented either horizontally or vertically.

On this page

  • When to use
  • What to avoid
  • Design and code
  • Content and design guidelines
  • Accessibility guidelines

When to use

  • To show the relative size or value of different data points
  • To compare the values of different data points
  • To show the distribution of data over a range of values

Examples

  • A bar graph comparing the vaccination rates of different age groups in a population
  • A bar graph showing the number of people who have been screened for a specific health condition, broken down by gender

When to use vertical bar charts

  • You have a large number of data points or categories
  • You want to emphasize the y-axis values
  • You want to show the data in a compact, space-efficient way

When to use horizontal bar charts

  • You have long category labels
  • You want to emphasize the x-axis values
  • You want to show the data in a way that is easy to read and compare

What to avoid

  • Don't use bar charts to show data that does not have a clear numeric value
  • Don’t use bar charts to show data with different units or different scales

Content and design guidelines

  • Always start the Y axis at 0
  • Consider ordering the bars by value
  • Use colour wisely - in general, use the same colour for all bars, and use colour to highlight a specific bar
  • Consider having the labels within the bars, instead of on the axis
  • Design bars so that they are wider than the space between them
  • Follow the general design guidelines

Accessibility guidelines

  • Provide a data table with the graph
  • If the bar graph tells a clear story, include that story in the alt text - for example, if the graph shows a clear increase over time, spell that out in the alt text
