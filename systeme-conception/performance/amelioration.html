Une mentalité numérique

Nous devons nous éloigner de la mentalité « lancer et oublier ». Adopter une mentalité numérique (et non pas une mentalité issue du monde de l'imprimé), où rien n'est jamais final, est crucial.

Même si la recherche et les tests sont bien effectués, certains problèmes ou améliorations potentielles peuvent ne surgir qu'après le lancement. Le travail n'est jamais « terminé ».

Commencer par le cadre de mesure de la performance

Si vos indicateurs de performance ne sont pas satisfaisants, utilisez des preuves pour améliorer votre produit.

Obtenez des preuves et améliorez

De nombreuses sources de données peuvent être utilisées, notamment:

  • une évaluation heuristique
  • de l'analyse web
  • la rétroaction des utilisateurs
  • les tests de convivialité

Évaluation heuristique

Une évaluation heuristique est une manière d'évaluer un produit numérique ou une interface sur la base d'un ensemble de principes d'utilisabilité. En général, l'évaluation est effectuée par plusieurs personnes et évalue le produit sur des principes tels que:

  • le contenu du produit est-il facile à trouver?
  • le contenu est-il clair?
  • le produit est-il connecté au reste de l'écosystème?
  • le produit est-il centré sur l'homme?
  • le produit est-il utile?

Analytique Web

Consultez l'analyse web pour comprendre les comportements actuels des utilisateurs sur votre produit.

Voir les métriques d'analyse web pour plus de détails.

Rétroaction des utilisateurs

Ajouter l'outil de rétroaction dans les produits clés peut être utile pour soutenir votre plan d'amélioration continue.

  • Outil de rétroaction sur la page

La rétroaction des utilisateurs ne devrait pas être utilisée comme indicateur de performance, mais plutôt comme des données qualitatives pour aider à améliorer le produit. Elle n'est utile que si les propriétaires du produit s'engagent à consulter régulièrement les commentaires et à agir en conséquence.

Ajout de l'outil de rétroaction

Pour ajouter l'outil de retour à une page, vous devez remplacer le widget Signaler un problème habituel et le bouton Partager avec un code HTML, puis ajouter un script JS.

Mettez à jour les 2 champs suivants dans HTML avec les informations de la page:

  • Titre de la page
  • URL de la page

Tests de convivialité

Évaluation formative vs évaluation sommative

Les évaluations formatives et sommatives diffèrent par leur objectif :

"Les évaluations formatives sont utilisées dans un processus itératif pour apporter des améliorations avant la production. Les évaluations sommatives sont utilisées pour évaluer un produit expédié par rapport à une référence."

Source: NN Group (en anglais seulement)

Évaluations formatives

Les évaluations formatives sont utilisées pendant la conception d'un produit pour l'améliorer de façon itérative et s'assurer qu'il fonctionne réellement. C'est une partie du processus de design.

Aussi peu que 3-5 participants peuvent suffire à découvrir des problèmes de convivialité importants et à apprendre comment les gens interagissent avec le produit. Pendant la phase de conception, il est préférable de faire plusieurs rondes de 5 participants plutôt qu'une ronde de 15 participants.

En raison du faible nombre de participants, il n'est pas possible de rapporter les résultats des tests en termes de pourcentage - les chiffres ne sont pas assez fiables. Ce genre de test peut être effectué rapidement à l'aide d'outils en ligne et n'est pas très intensif en termes de ressources.

  • Why You Only Need to Test with 5 Users (NN Group) (en anglais seulement)
Évaluations sommatives

Les évaluations sommatives sont utilisées pour mesurer la réussite des tâches. Il faut tester une référence, puis retester le contenu après des modifications.

Elle nécessite 15 participants ou plus - idéalement 16 à 18. Avec un grand nombre de participants, les résultats sont généralement reproductibles. Vous pouvez rapporter la réussite en pourcentage. Il peut être utilisé pour suivre les performances dans le temps.

L'évaluation sommative est plus intensive en termes de ressources que l'évaluation formative'.

Écriture des scénarios de tâches

Utilisez les recherches effectuées pour comprendre les tâches de données pour informer vos scénarios de tâches.

Consultez les ligne directrices du Bureau de la transformation numérique pour vous aider à écrire de bons scénarios de tâches.

  • Amélioration continue - Phase exploratoire (BTN)

Modéré vs non modéré

Les tests modérés sont à privilégier dans les situations suivantes :

  • pour un processus ou un concept compliqué
  • lorsque vous voulez poser des questions complémentaires lorsqu'un participant est bloqué ou confus
  • pour travailler avec un groupe de participants spécialisé

Les test non modérés sont à privilégier dans les situations suivantes :

  • pour des résultats rapides
  • pour étudier quelques tâches très spécifiques
  • si vous avez seulement besoin de sélectionner les participants en utilisant des informations démographiques de base (âge ou revenu)
